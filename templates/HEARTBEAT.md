# Heartbeat Checklist（心跳检查清单）

> 每次心跳唤醒时执行以下流程。这是 SCL R-CCAM 的完整一轮。

---

## ⚡ Pre-Check：目标就绪检查（最优先）

**在进入 R-CCAM 循环之前，先执行此检查。**

读取 `TASKS.md` 中的「顶层目标」字段：

```
如果「顶层目标」为空或是模板占位符（如「[在此填入顶层目标]」）：
  → 说明尚未设定目标，不执行后续步骤
  → send_message：
    「我已就绪，请告诉我你的目标。
     格式：目标：[你的目标描述]
     例如：目标：帮我在三个月内让小红书账号涨粉 1000 人」
  → 本次心跳结束，等待用户输入目标

如果「顶层目标」存在且非占位符：
  → 继续执行 R-CCAM 循环
```

---

## R — Retrieval（读取当前状态）

1. 读取 `TASKS.md`：
   - 确认顶层目标
   - 找到当前应执行的子任务节点（`[~]` 进行中 或 下一个 `[ ]`）
   - 检查是否有「建议重分解」标记

2. 快速扫描 `MEMORY.md`：
   - 确认当前约束条件
   - 查看最新的领域知识（可能影响本轮策略）

3. 查看今天的 Daily Log（若已有内容）：
   - 本轮之前是否已执行过？避免重复

---

## C — Cognition（决策本轮行动）

1. **若 TASKS.md 中没有任务树**（首次运行或新目标）：
   - 调用 `recap-decompose` Skill，生成子任务树
   - 将任务树写入 TASKS.md，标记第一个节点为 `[~]`
   - 本轮行动：执行第一个节点

2. **若任务树存在且有待执行节点**：
   - 选择当前 `[~]` 节点（或第一个 `[ ]` 节点）
   - 确定本轮具体行动（对应节点的完成标准）

3. **若当前层级全部完成**（所有节点为 `[x]` 或 `[!]`）：
   - 调用 `recap-decompose` Skill，进行重分解
   - 基于已完成内容和实际效果，生成下一层任务树

4. **若 HEARTBEAT_OK 条件满足**（无需行动）：
   - 无紧急任务，今天已执行 N 次，无新情况
   - 回复 `HEARTBEAT_OK`，本次心跳结束

---

## C — Control（执行前验证）

调用 `scl-control` Skill，验证本轮计划行动：
- 范围是否符合当前子任务？
- 是否违反 MEMORY.md 中的约束？
- 所需工具是否可用？（若不可用，进入能力缺口自举）

---

## A — Action（执行）

执行通过 Control 验证的行动。
- 每次心跳只执行一个子任务节点
- 若行动涉及多个步骤，完成所有步骤后再进入 M 阶段

---

## M — Memory（持久化状态）

调用 `scl-memory` Skill：
1. 将本轮行动和结果写入 Daily Log
2. 更新 TASKS.md 中的任务节点状态
3. 若是第 5N 次心跳，提炼领域知识到 MEMORY.md

---

## 心跳结束判断

- 若行动成功：正常结束，等待下次心跳
- 若发现能力缺口：capability-gap-handler 处理后结束（新 Skill 下次心跳生效）
- 若遇到真正阻塞：send_message 通知用户，等待响应
- 若无需行动：回复 `HEARTBEAT_OK`，静默结束
