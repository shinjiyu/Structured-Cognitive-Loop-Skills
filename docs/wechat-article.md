# AI Agent 的范式跃迁：为什么「心跳」比「计划」更重要

**你有没有发现一件事——**

我们用 AI 写代码、做任务，总是在做同一件事：

**先让 AI 想好计划，再让它执行。**

这个思路如此自然，以至于没人质疑过它。

但最近，有人用一行 bash 命令，无意间打开了一扇门。

---

## 第一代：先计划，再执行

这是我们最熟悉的模式。

打开 Claude 或 Cursor，描述需求，让 AI 列出步骤，然后按步骤执行。

Plan 是核心产物，越详细越好，一次完成越多越好。

**但这个模式有一个隐藏的脆弱性：**

- 执行到一半被打断？计划作废，从头来
- 世界变了，计划没变？越执行越跑偏
- 目标太抽象（比如「帮我运营账号」）？第一步都不知道怎么写

计划越宏大，系统越脆弱。

---

## 第二代：Ralph Loop——一个意外的启示

2025 年，工程师 Geoffrey Huntley 发布了一个叫「Ralph」的技术。

它的全部实现，就是这一行：

```bash
while :; do cat PROMPT.md | claude-code ; done
```

把 AI 套进一个无限 bash 循环。任务被打断了？下一轮自动重启。状态通过文件持久化。

**Ralph 的本意很简单：解决「任务被打断后要手动重启」的麻烦。**

但它碰巧暗示了一件更深刻的事：

> 如果 Agent 天然就会被反复唤醒，「不追求单次长时间运行」就变成了一个合理的选择。

这不是 Ralph 的设计目标，只是它结构上的副作用。但这个副作用，指向了一个全新的方向。

---

## 第三代：Heartbeat——AI 真正「活着」了

OpenClaw 的 Heartbeat（心跳）机制，是对这个方向的第一个严肃实现。

它的运行模式是这样的：

```
持久化的【目标】 + 持久化的【现状】
        ↓ 心跳唤醒（每 N 分钟）
  选择当前最优的单步行动
        ↓
  执行 → 把结果写回现状
        ↓ 下次心跳唤醒
  选择当前最优的单步行动
        ↓
  无限重复……
```

看起来和 Ralph 很像，但有一个根本性的不同：

**Ralph 是为了「不断重试一个任务」；Heartbeat 是为了「永远向目标逼近」。**

Heartbeat 模式下，Agent 每次被唤醒，只做一件极简单的事：

> 我现在在哪？我要去哪？当前最优的一步是什么？走它。

没有宏大的计划，没有复杂的编排。只有目标、现状、和一步行动。然后下次心跳，再来一次。

---

## 一个反直觉的结论

Plan-Execute 模式和 Heartbeat 模式，有一个让人意外的差异：

|  | Plan-Execute | Heartbeat Loop |
|--|:---:|:---:|
| 核心产物 | 完整的 Plan | 准确的现状（State） |
| 被打断后 | 一切作废 | 下次心跳继续 |
| 容错性 | 随时间**递减** | 随时间**递增** |
| 对 Agent 的要求 | 强规划能力 | 强状态感知能力 |

**容错性随时间递增**——这是最反直觉也最重要的一点。

传统模式里，运行越久，小错误积累越多，系统越容易崩。

Heartbeat 模式里，每次心跳都是一个独立的、从头开始的决策。上一轮出错，这一轮重新看清楚现状，自然纠偏。系统不会「崩溃」，只会「暂时偏离」，然后被下一次心跳拉回来。

**活着的系统，不需要完美的计划。它只需要一直知道自己在哪，要去哪。**

---

## OpenClaw 被低估的部分

很多人使用 OpenClaw，关注的是它的 Skill 系统有多丰富、Memory 系统有多强大。

但我认为，**Heartbeat 才是 OpenClaw 最有价值的部分**，也是被最低估的部分。

没有 Heartbeat，Skill 和 Memory 只是对旧范式的功能增强。

有了 Heartbeat，它们的角色完全改变：

- **SOUL.md** → 定义 Agent 的决策偏好和约束
- **TASKS.md** → 结构化记录当前目标和现状
- **MEMORY.md** → 跨时间的长期状态压缩
- **Skills** → 每次心跳可调用的行动集合
- **HEARTBEAT.md** → 每一轮决策的执行 SOP

这些组件合在一起，构成了一个真正意义上「活着的 Agent」：有目标、有记忆、有行动力，能在无人监督的情况下，一次心跳接着一次心跳，自主向目标逼近。

## 我们做了什么

基于这个范式认识，我们设计了一套可以安装到 OpenClaw 上的 Skill 系统：

**Structured Cognitive Loop Skills**

它把「心跳循环」在认知科学层面形式化，分解为五个可追溯的模块（R-CCAM）：

- **goal-setting**：目标是 Loop 的锚点，显式设定、持久化、写入验证
- **scl-control**：每次行动前验证是否符合约束，防止越界
- **scl-memory**：每次心跳结束后结构化写入现状，State 质量是系统命脉
- **recap-decompose**：把抽象目标分解为有限任务树，缩小每轮的搜索空间
- **capability-gap-handler**：发现能力缺口时，自动用已有工具派生新能力

还附带了完整的配置文件模板（SOUL.md / HEARTBEAT.md / TASKS.md / MEMORY.md）和详细的安装指南。
